# 투자 보조 프롬프트

국내·미국 주식(ETF 제외) 매수·매도 판단을 보조하기 위한 AI 챗 프롬프트 모음. 웹 검색 가능한 환경(ChatGPT, Claude, Gemini 등)에서 사용하는 것을 전제로 한다.

## 구성

| 파일 | 용도 |
|---|---|
| [`prompts/stock-analysis.md`](prompts/stock-analysis.md) | 종목명·티커 입력 → 종목 분석 |
| [`prompts/macro-liquidity.md`](prompts/macro-liquidity.md) | 시점 입력 → 시장·거시·유동성 진단 |
| [`PLAN.md`](PLAN.md) | 두 프롬프트의 설계 원칙·구조 정리 |

## 사용법

1. 사용할 프롬프트 파일을 연다.
2. `=== 프롬프트 시작 ===` 부터 `=== 프롬프트 끝 ===` 까지를 통째로 복사한다.
3. 웹 검색 가능한 AI 챗에 붙여 넣는다.
4. 마지막 줄의 변수를 채운다.
   - 종목 분석: `{{TICKER_OR_NAME}}` → `NVDA`, `삼성전자` 등
   - 시장 분석: `{{WHEN}}` → 비워도 됨, 또는 `오늘`, `이번주`, `2026-04-29` 등
5. 전송한다.

### 입력 예시

종목 분석:
```
분석 대상: NVDA
```
```
분석 대상: 삼성전자
```

시장 분석:
```
시점: 오늘
```
```
시점: 2026-04-29
```

## 출력 공통 규칙

- **헤더 결론** 3~5줄로 핵심을 먼저 보여준다 (매수/매도/관망 또는 시장 국면 + 한 줄 근거).
- 본문은 표 중심으로 A4 1.5장 분량.
- 모든 수치 옆에 **출처를 클릭 가능한 링크**(`[FRED](URL)` 같이)와 **조회 날짜**가 붙는다.
- 모르거나 데이터가 오래되면 **"미확인"** 으로 표기. 추정·창작 금지.
- 업종상 맞지 않는 밸류에이션 지표는 **"부적합"** 으로 표기한다.
- 정책·규제의 숫자 영향은 근거가 있을 때만 쓰고, 없으면 **"정량 추정 불가"** 로 표기한다.
- 진입가·손절가·목표가·적정가는 지지/저항, 이동평균, 컨센서스, 밸류에이션 같은 근거를 반드시 붙인다.
- 본문은 한국어, 티커·영문 고유명사·숫자·단위는 원문 유지.
- **스윙(수주~수개월)** 과 **중장기(1년+)** 의견을 분리해서 출력. 두 시간축이 상충하면 그 이유 명시.
- 매크로·유동성 표에는 각 지표마다 **한 줄 설명**과 **"더 알아보기" 학습용 링크**가 붙는다.
- 마지막에 `## 한 줄 결론` 강제.

## 어떤 프롬프트를 언제 쓰는가

| 상황 | 추천 프롬프트 |
|---|---|
| 특정 종목을 살까 말까, 팔까 말까 | 종목 분석 |
| 보유 종목의 펀더멘털·경쟁력 점검 | 종목 분석 |
| 지금 시장이 어떤 국면인지 모르겠을 때 | 시장 분석 |
| 비중 조절·헤지 필요 여부 판단 | 시장 분석 |
| 새 자금 투입 타이밍 검토 | 시장 분석 → 종목 분석 순서로 |

## 한계·면책

- 이 프롬프트의 출력은 **투자 판단 보조**에 불과하다. **최종 책임은 사용자**에게 있다.
- AI 챗은 실시간 호가·체결 데이터에 접근할 수 없거나 부정확할 수 있다. **매매 직전엔 HTS/MTS로 재확인**할 것.
- 거시 지표는 발표 시차(특히 GDP·고용 수정치)가 존재한다. 출력의 **조회 날짜를 반드시 확인**할 것.
- 정책·규제 분석은 AI가 최신 입법 동향을 놓칠 수 있다. 의심되면 1차 자료(법안 원문, 부처 보도자료)를 직접 확인.
- 동일 종목에 대해서도 모델·시점·검색 결과에 따라 답이 달라질 수 있다. 한 번의 출력으로 단정짓지 말 것.

## 갱신 주기

분기 1회 검토 권장. 검토 시 점검할 것:
- 매크로·유동성 표의 지표 추가/삭제 (시장 국면 변화에 따라)
- 데이터 소스 URL이 깨지거나 정책이 바뀌었는지
- 정책·규제 섹션의 예시(CHIPS Act, IRA, 밸류업 등)가 현 시점에도 유효한지
- 표 컬럼·우선순위 조정 필요 여부
